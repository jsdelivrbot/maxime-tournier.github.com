A short summary of the steps needed to get a working development environment for kinect on unix:

  • open-source kinect driver: libfreenect from OpenKinect
  • OpenNI2 framework, with the Freenect driver
  • NiTE2 skeleton tracker
  • python bindings

1 OSX

1.1 Homebrew/Cask

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
brew install caskroom/cask/brew-cask

1.2 Java

brew cask install java

1.3 tap homebrew/science

brew tap homebrew/science

1.4 OpenNI2

brew install homebrew/science/openni2

1.5 Freenect driver for OpenNI2

1.5.1 Homebrew

do this if you don't care about using fakenect with openni.

1.5.1.1 libfreenect
brew install libfreenect
1.5.1.2 openni2-freenectdriver
brew tap totakke/openni2
brew install openni2-freenectdriver

# Add the recommended variables to your dotfiles.
# ,* On Bash, add them to `~/.bash_profile`.
# ,* On Zsh, add them to `~/.zprofile` instead.

export OPENNI2_INCLUDE=/usr/local/include/ni2
export OPENNI2_REDIST=/usr/local/lib/ni2

Openni2-Freenectdriver links to libfreenect statically, so you won't be able to use fakenect with it if you don't patch it.

1.5.2 Sources

If you want to use fakenect with OpenNI2 (e.g. for skeleton tracking on recorded sequences), use this method instead.

I had to patch fakenect to make it work, because I could not use OniFile recorder to do skeleton tracking (probably calibration/registration related). And OpenNI2 seems half-dead anyways.

The patch adds stubs to fakenect where needed, and converts DEPTH_11BIT to DEPTH_MM if this format is requested (it is by NiTE2 skeleton tracker).

1.5.2.1 clone libfreenect

In this case, the one with my patches: 

git clone https://github.com/maxime-tournier/libfreenect.git
1.5.2.2 configure
cd libfreenect
mkdir build
cd build
cmake -L .. -DCMAKE_BUILD_TYPE=Release -DBUILD_OPENNI2_DRIVER=ON
1.5.2.3 build and install
make install
1.5.2.4 symlink driver
ln -s /usr/local/lib/OpenNI2-FreenectDriver/libFreenectDriver.dylib /usr/local/lib/ni2/OpenNI2/Drivers
